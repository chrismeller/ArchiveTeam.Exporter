# Remove Dead GZip Handling from ProjectService

## Change

Two edits in `src/ArchiveTeam.Exporter.ApiService/Services/ProjectService.cs`:

### 1. Remove `using System.IO.Compression;` (line 3)

No longer needed after removing the GZip branch.

### 2. Replace lines 66–78 (the byte-array read + conditional decompression) with a single string read

```csharp
// Before:
var bytes = await response.Content.ReadAsByteArrayAsync(cancellationToken);
string json;

if (response.Content.Headers.ContentEncoding.Contains("gzip"))
{
    using var gzipStream = new GZipStream(new MemoryStream(bytes), CompressionMode.Decompress);
    using var reader = new StreamReader(gzipStream);
    json = await reader.ReadToEndAsync(cancellationToken);
}
else
{
    json = System.Text.Encoding.UTF8.GetString(bytes);
}

// After:
var json = await response.Content.ReadAsStringAsync(cancellationToken);
```

The `HttpClientHandler.AutomaticDecompression` in `Program.cs` transparently decompresses the response and strips the `Content-Encoding` header, so the manual `GZipStream` branch is dead code.
