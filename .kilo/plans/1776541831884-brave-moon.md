# Simplify ProjectService — Two Methods, No State

## Goal

`ProjectService` should be a stateless, transient service with two public methods:

1. **`FetchProjectsAsync`** — calls the HTTP endpoint, deserializes the JSON, returns `ArchiveTeamProject[]`
2. **`GetProjectGaugesAsync`** — calls `FetchProjectsAsync`, then publishes the Prometheus gauge labels from the results and returns the gauges (or just the projects)

## Changes

### `IProjectService.cs`

```csharp
Task<ArchiveTeamProject[]> FetchProjectsAsync(CancellationToken cancellationToken);
Task<ArchiveTeamProject[]> GetProjectGaugesAsync(CancellationToken cancellationToken);
```

### `ProjectService.cs`

Strip everything down:

- **Remove**: `_projects` field, `_previousLabels` HashSet, `SanitizeLabel()`, `GetProjects()`, `IDisposable`
- **Keep**: `HttpClient`, `IConfiguration`, `ILogger`, `_projectInfoGauge` (still needed for gauge publishing)
- **`FetchProjectsAsync`**: fetch URL → deserialize → return the array. No gauge logic, no state storage.
- **`GetProjectGaugesAsync`**: calls `FetchProjectsAsync`, iterates the projects, publishes gauge labels with `SanitizeLabel`, returns the projects.

Target: ~60-70 lines total, zero mutable state between calls.

### `Program.cs`

Update `/metrics` endpoint to call `GetProjectGaugesAsync` instead of `FetchProjectsAsync`.
