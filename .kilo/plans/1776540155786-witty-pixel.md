# Remove Automated Refresh from ProjectService

## Goal
Remove the periodic timer-based refresh in `ProjectService` and instead fetch fresh project data on each `/metrics` scrape. Cache only the label tracking (`_previousLabels`) between calls for proper Prometheus gauge cleanup.

## Changes

### 1. `ProjectService.cs` — Convert from BackgroundService to regular service
- **Remove** `BackgroundService` base class and `ExecuteAsync` override
- **Remove** `PeriodicTimer` and the refresh interval configuration logic
- **Remove** the `IHostedService`-related registration (no longer a background service)
- **Make `FetchProjectsAsync` public** so it can be called on-demand from the metrics endpoint
- Keep the class as a regular service implementing `IProjectService`
- Keep the `SemaphoreSlim`, `_previousLabels`, `_projectInfoGauge`, and all gauge label management logic intact

### 2. `Program.cs` — Replace `MapMetrics()` with custom metrics endpoint
- **Remove** `builder.Services.AddHostedService<ProjectService>()`
- **Register** `ProjectService` as a singleton (it's already registered via `AddHttpClient<ProjectService>`, but remove the hosted service registration)
- **Replace** `app.MapMetrics()` with a custom endpoint at `/metrics` that:
  1. Calls `projectService.FetchProjectsAsync()` to fetch fresh data
  2. Uses the Prometheus library's `CollectorRegistry` to render metrics output
- This ensures every Prometheus scrape gets fresh project data

### 3. `appsettings.json` — Remove refresh interval config
- **Remove** `RefreshIntervalMinutes` from the `ArchiveTeam` config section (no longer needed)

## Summary
The service shifts from "periodically fetch in background, serve cached data on scrape" to "fetch fresh data on each scrape, cache only label tracking between calls."
