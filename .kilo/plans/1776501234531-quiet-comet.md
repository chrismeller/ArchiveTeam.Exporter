# Plan: Prometheus /metrics Endpoint & ArchiveTeam Project Service

## Overview

Add a Prometheus `/metrics` endpoint to the existing `ArchiveTeam.Exporter.ApiService` project and create a service class that fetches and parses the list of active projects from `https://warriorhq.archiveteam.org/projects.json`.

## Data Model (from live JSON)

Each project in the JSON has this shape:
```json
{
  "name": "telegram",
  "title": "Telegram",
  "description": "Archiving Telegram. Long term project...",
  "repository": "https://github.com/ArchiveTeam/telegram-grab",
  "leaderboard": "https://tracker.archiveteam.org/telegram",
  "logo": "https://wiki.archiveteam.org/images/...",
  "lat_lng": [0, 0],
  "marker_html": "<a href='...'>...</a>"
}
```

## Steps

### 1. Add `prometheus-net` NuGet package to ApiService

Add `prometheus-net` (and `prometheus-net.AspNetCore`) to `ArchiveTeam.Exporter.ApiService.csproj`. This is the standard .NET Prometheus library that provides the `/metrics` endpoint and the `Metrics` builder API.

### 2. Create data models

Create `Models/ArchiveTeamProject.cs` in the ApiService project with a class matching the JSON shape:
- `Name` (string)
- `Title` (string)
- `Description` (string)
- `Repository` (string?)
- `Leaderboard` (string?)
- `Logo` (string?)

### 3. Create `IProjectService` interface and `ProjectService` implementation

Create `Services/IProjectService.cs` and `Services/ProjectService.cs`:
- `ProjectService` will be a `BackgroundService` that periodically fetches `https://warriorhq.archiveteam.org/projects.json` on a timer (configurable interval, default 5 minutes)
- It will deserialize the JSON and expose the project list via a public property/method
- It will register a Prometheus `Gauge` metric (e.g., `archiveteam_projects_info`) with labels for project name, title, description — value set to 1 for each active project
- The service URL will be configurable via `appsettings.json`

### 4. Wire up `Program.cs`

- Register `ProjectService` as a hosted service
- Add Prometheus middleware via `app.MapPrometheusMetrics()` at `/metrics`
- Remove the sample weather forecast code
- Keep the root endpoint and health checks

### 5. Update `appsettings.json`

Add configuration for:
- `ArchiveTeam:ProjectsUrl` → `https://warriorhq.archiveteam.org/projects.json`
- `ArchiveTeam:RefreshIntervalMinutes` → `5`

### 6. Verify the solution builds

Run `dotnet build` to confirm everything compiles.
